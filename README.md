Saya menggunakan 'payable' sebagai interface untuk class SalariedEmployee, PartTimeEmployee, dan Invoice. Karena mereka mempunyai kesamaan yaitu dapat dihitung jumlah yang harus di bayar menggunakan method getPaymentAmount(), tetapi class-class tersebut juga mempunyai perbedaan yaitu tidak semua satu inheritance (keluarga), contohnya employee inherit ke SalarieEmployee dan PartTimeEmployee, tetapi untuk invoice tidak inherit employee tersebut.Dengan interface, ketiga class tersebut bisa dianggap seperti objek bertipe 'payable' yang membuat adanya penerapan polymorphism lewat List<Payable> yang terjadi saat method getPaymentAmount di panggil di class-class tersebut mempunyai hasil yang berbeda sesuai implementasinya.Inheritance juga digunakan di class Employee yang dijadikan abstract class (blueprint setengah jadi) yang merepresentasikan hubungan 'is-a', yang dimana SalariedEmployee dan ParTimeEmployee menjadi inherit atau child class dari Employee dalam hierarki inheritance.
